Business Multi-User Data Isolation

Your business account supports per-user data isolation. Pass the X-Sub-User header with a unique identifier for each end-user. Bots created with a sub-user ID are only visible to requests using the same identifier.

Example: Create a bot scoped to a user

curl -X POST {{ request.base_url }}api/v1/bot \
  -H "Authorization: Bearer sk_live_..." \
  -H "X-Sub-User: user-123" \
  -H "Content-Type: application/json" \
  -d '{"meeting_url": "https://meet.google.com/abc-defg-hij"}'

Example: List only that user's bots

curl {{ request.base_url }}api/v1/bot \
  -H "Authorization: Bearer sk_live_..." \
  -H "X-Sub-User: user-123"

Omit the X-Sub-User header to see all bots across all sub-users (account-wide view).
